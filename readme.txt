1. server.js
первым делом запускается файл server.js, он отвечает за логику взаимодействия клиента с сервером, все комменты прописаны уже
строки 35-54 отвечают за определение типов запросов к сервуру (если где-то в js файлах будет отправлен запрос, то в этих строках определится, какой тип у этого запроса, и какой метод нужно вызвать)
например, если где-то вызвали post запрос для "/users", то в строке 51 определится, что данному запросу соответствует метод UsersController.create и этот метод будет вызван

2. модели (meal, menu, user)
в этих файлах описываются модели базы данных, их струкруры. так программа будет понимать, с какими свойствами нужно создавать объекты и какие свойства можно прочитать у уже созданных объектов

3. контроллеры 
в этих файлах описывается логика взаимодействия с моделями базы данных
в каждом файле есть методы индекцсации(определения)/создания/обновления/удаления/отображения
эти методы обращаются к json-файлам, содержащим объекты конкретной модели и работают с ними

папка клиента:

index.html - главная страница сайта (страница гостя)
логика работы страницы описана в guest.js

authorization.html - страница авторизации в систему
логика работы страницы описана в users.js

admin.html - страница админа
логика работы страницы описана в admin.js

worker.html - страница работника
логика работы страницы описана в worker.js
